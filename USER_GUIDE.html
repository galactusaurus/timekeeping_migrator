‚öôÔ∏èWhat This Tool Does

The Timekeeping Migrator is an automated tool that:

  • Reads time entry data from your Access database
  • Filters the data based on date range you specify
  • Converts the data to a standard SQLite format
  • Exports the results as a CSV file for easy viewing and sharing
üí° In Simple Terms: Think of it like taking a snapshot of your time entries from a specific date range and saving it in a format that Excel and other tools can easily read.

‚öôÔ∏èConfiguration Setup

Before running the script, you need to configure the config.yaml file with your settings.

How to Edit config.yaml

  1. Find and open the config.yaml file in this folder
  2. Right-click and select "Edit with Notepad" or your preferred text editor
  3. Update the settings below
  4. Save the file

Required Parameters

path_to_access_db
The location of your Access database file (.accdb file)
Format: C:\\Path\\To\\Your\\Database.accdb
Example: C:\\testData\\AGE-Projects_be.accdb
Note: Make sure to use double backslashes (\\) in the path
start_date
The beginning date for your export range
Format: MM-DD-YYYY
Example: 01-15-2025 (January 15, 2025)
Note: Only time entries on or after this date will be exported
end_date
The ending date for your export range
Format: MM-DD-YYYY
Example: 12-31-2025 (December 31, 2025)
Note: Only time entries on or before this date will be exported
query_file_used
The SQL query file that defines what data to export
Default: .\\BQE_Export_Query.sql
Note: Leave this as-is unless instructed otherwise

Example Configuration

# Configuration file for timekeeping migrator

##### Access Export Parms
path_to_access_db: "C:\\testData\\AGE-Projects_be.accdb"
start_date: "01-15-2025"
end_date: "12-31-2025"

##### CSV Builder Parms
sqlite_database_path: ""
query_file_used: ".\\BQE_Export_Query.sql"
‚ö†Ô∏è Important: Don't forget to save the config.yaml file after making changes! The script will use whatever settings are saved when you run it.

üîßTroubleshooting

Common Issues

Error: "Python is not installed or not in PATH"
Solution:
  • Download Python 3.8 or higher from python.org
  • During installation, check the box "Add Python to PATH"
  • Restart your computer
  • Try running the script again
Error: "config.yaml not found"
Solution:
  • Make sure the config.yaml file exists in the same folder as ExportTimeEntries.bat
  • Don't rename or move the file
  • If it's missing, contact your administrator
Error: "Export to SQLite failed"
Solution:
  • Check that the path in config.yaml points to an existing Access database file
  • Make sure the file isn't open in another program
  • Verify your date range (start_date should be before end_date)
The script is very slow or seems frozen
Solution:
  • Large databases can take several minutes to process
  • If your database has a lot of data, the script may take 5-15 minutes
  • Don't close the window - let it continue running

üíªSystem Requirements

  • Operating System: Windows 7 or later

  • Disk Space: At least 500 MB free

üìù Note: The script will check all requirements and let you know if anything is missing before starting the export process.
